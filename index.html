Linkage Bandit guide

How to use this workspace

1. Load and configure data

Choose a CSV or lossless JSON file for Data A and Data B. CSV imports ask for a delimiter; choose Auto-detect unless you know the file uses a different separator. Open Workspace settings to select visible fields, filters, sorting, and the matching field on each side.

2. Read and create relationships

Black lines are pre-existing links inferred from the selected matching fields. Blue lines are new linkages recorded in changes. Click one card on each side and use the Add new linkage hotkey. Select a blue line, then use Delete selected to remove it.

Linkage layout can group a child side under an anchor side. Connection space gives the lines room to breathe. Hide/show changes controls every blue change layer; hide/show pre-existing linkages controls black inferred links.

3. Edit records safely

Select a single card and use Edit record. Only displayed fields appear in the editor. Edited values are blue. Add record creates a light-blue record on either side; Add linked record creates one on the opposite side and adds a blue link automatically. Delete selected removes a new record after confirmation, or marks a source record for deletion without modifying the source CSV.

4. Save, export, and recover

Save changes downloads a .changes sidecar containing edits, new records, deletions, and new linkages. Save config downloads a .config sidecar containing workspace settings and hotkeys. Use Undo/Redo for recent in-session changes. The app warns before leaving when changes have not been saved.

Integrated export applies the change layer to CSV or lossless JSON. It also writes the selected linkage field from linked opposite-side records. One linked record becomes a single value; multiple linked records become a Python-style list. Confirm the selected linkage fields in the export warning before continuing.

Errors and warnings

  • Unsupported schema: the sidecar is not a supported changes-v1 or relationship-config-v1 file.
  • CSV needs a header row: the import has no usable column headings or data rows.
  • Unclosed quoted value: a quoted CSV cell is missing its closing quote.
  • Custom delimiter required: choose one character before loading a custom-delimited CSV.
  • Configuration expecting…: a config remembers a filename that has not been selected in this browser session. This is non-blocking.
  • Ghost records / unresolved changes: a sidecar references a row absent from the current dataset. Inspect or dismiss it from the Changes panel.

Known limitations

  • Browsers cannot silently reopen arbitrary local files, so loading a config cannot automatically load its CSVs or changes sidecar.
  • Record identity is based on the complete original row. Exact duplicate rows can therefore be ambiguous for manual links and changes.
  • Very large datasets or many-to-many relationship sets can make the connector view crowded or slower.
  • Auto-detect supports comma, semicolon, tab, and pipe. Other separators require the custom option.
  • This is a local, single-user workspace. It does not merge competing sidecars or provide real-time collaboration.

Hotkeys

Open the floating Hotkeys panel to view or record shortcuts. Undo and Redo defaults are Ctrl+Z and Ctrl+Y. Workspace settings and hotkey choices are saved in config.
